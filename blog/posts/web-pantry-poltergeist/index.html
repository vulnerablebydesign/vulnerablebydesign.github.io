Three Drawers, One Trick

The brief: "Legacy ops bookkeeping scattered three fragments of the vault passphrase across browser storage tiers when the page loaded. Reassemble them in order."

"Storage tiers" is the puzzle's marketing copy. The challenge wants you to inspect localStorage, sessionStorage, and cookies and assemble three pieces in order. That's the textbook walkthrough.

The actual exploit is one grep.

The Three Drawers Are The Same JS File

I had already pulled /admin/challenge-logic.js for the Freezer Aisle (stage 01) recon — same file holds the handlers for all six Web stages. The Pantry Poltergeist handler is staring at you in plain text:

// 02 Pantry Poltergeist // Three fragments written to localStorage, sessionStorage, and cookies // on page load. Player reassembles and submits. try { localStorage.setItem('vault.part1', 'R2-MUNCHIES-'); sessionStorage.setItem('vault.part2', '2025-HRC-'); document.cookie = 'vault_part3=LOUISVILLE; path=/admin/; SameSite=Strict'; } catch (e) { ... }

The "secret" the storage tiers were "hiding" is literal string content of the JavaScript file the server is happy to hand any visitor. Concatenated in declared order:

R2-MUNCHIES- + 2025-HRC- + LOUISVILLE = R2-MUNCHIES-2025-HRC-LOUISVILLE

(Lore aside: that's "Round 2 — Munchies — 2025 HRC Louisville." HRC = Hack Red Con, a security conference held in Louisville KY in fall 2025. The challenge encodes a real-world conference reference.)

What I Learned

Browser storage APIs are the same trust boundary. localStorage, sessionStorage, and cookies feel different — different scopes, different lifetimes, different APIs — but they're all client-side. Whatever code writes to them runs in the browser, which means anyone with view-source can read the writers and back-derive the values. The "three places" framing is performative complexity.

The fix is server-side derivation. If a value needs to be unguessable, the server needs to be the only entity that ever knew it. The pattern this challenge is satirizing is "developer pre-computes the secret in JS to save a round-trip" — which is functionally publishing the secret.
