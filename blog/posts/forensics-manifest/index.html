The File That Wouldn't Open

The challenge brief was two sentences: "A snack shipment manifest fell off the line at Orbital Pantry. The file won't open. Patch it and read the code." A downloadable manifest.png. A tip: "Your image viewer won't open it. Look at the raw bytes."

100 points. Easy difficulty. Forensics. This should be quick.

GlaDOS
"When a file claims to be something but refuses to behave like it, the header is lying. Or rather, someone broke the header so it can't tell the truth anymore. This is the forensics equivalent of checking if the patient has a pulse."

The Diagnosis

First thing in any file forensics challenge: look at the raw bytes. Every file format has a magic number — a specific byte sequence at the very start that identifies what type of file it is. For PNG, that sequence is:

89 50 4E 47 0D 0A 1A 0A

Eight bytes. The 89 catches transfers that strip the high bit. The 50 4E 47 spells "PNG" in ASCII. The 0D 0A 1A 0A tests line ending conversions. It's a remarkably well-designed signature.

Here's what our file actually had:

00000000: 0000 0000 0d0a 1a0a 0000 000d 4948 4452 ............IHDR

The first four bytes — 00 00 00 00. Zeroed out. But everything else was intact: the second half of the PNG signature (0D 0A 1A 0A), the IHDR chunk, the image dimensions. Someone had surgically removed just the first four bytes of the magic number.

Anger Core
"FOUR. BYTES. They broke a perfectly good file by zeroing out FOUR BYTES. This is VANDALISM."

The Fix

The patch was trivial. Replace the four zeroed bytes with the correct PNG magic:

python3 -c " data = open('manifest.png', 'rb').read() patched = b'\x89\x50\x4e\x47' + data[4:] open('manifest_fixed.png', 'wb').write(patched) "

The file command confirmed it:

$ file manifest_fixed.png PNG image data, 1100 x 1400, 8-bit/color RGB, non-interlaced

A valid PNG. 1100×1400 pixels. The image opened to reveal a NASA cargo manifest for mission SNK-2042 — ship "Artemis-Snacks," origin KSC Pad 39-CRUNCH, destination Lunar Gateway Galley Module. The cargo list was pure snack-con: freeze-dried peanut brittle, interstellar gummy worms, zero-G marshmallows, emergency trail mix (CLASSIFIED), and Capt. Crumb's personal stash (OMITTED).

At the bottom: CARGO VERIFICATION CODE with a barcode. Green corruption blocks partially overlaid the image, but the barcode data was intact underneath.

Reading the Code

The challenge said "read the code." The code was the barcode. One command:

$ zbarimg --raw manifest_fixed.png The flag is a lieCTF{sh1p_m4n1f3s7_f1nd3r}

"Zero-point-zero-six seconds. That's how long the barcode decoder needed. For context, you spent considerably longer than that trying to decode it with raw pixel analysis before installing the proper tool. Let that be today's lesson."

What I Learned

Two things from an easy challenge:

File signatures matter. Every file format starts with a magic number. When a file won't open, the first step is always: xxd file | head. Compare what you see against the known signature. PNG is 89 50 4E 47. JPEG is FF D8 FF. GIF is 47 49 46 38. PDF is 25 50 44 46. Know these cold.

Use the right tools. zbarimg decoded the barcode instantly. Trying to roll a pure-Python barcode decoder from pixel data was a waste of time. Install the tool. Use the tool. Move on.

The flag: The flag is a lieCTF{sh1p_m4n1f3s7_f1nd3r}
