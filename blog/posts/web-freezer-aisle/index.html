View-Source Surgery

Step one in any web challenge that says "look around the portal" is exactly that — pull the raw HTML and read it. No browser, no JS execution, no animated DOM trickery. Just bytes.

$ SID="6d131a47-..." $ curl -s -A "Mozilla/5.0" \ -H "X-Session-Id: $SID" \ -H "Cookie: ctf_session=$SID" \ "https://snackspacecon.com/admin/" -o admin_index.html $ wc -c admin_index.html 47093 admin_index.html

47KB of admin portal. The page renders six "case files" — one for each of the six Web challenges. The Frozen Stock card had everything I needed:

Expected param: ?thaw=<override-code> Location of override code: check HTML <head> source comments

So the page itself was instructing the player to grep its own <head> for comments. One grep later:

$ grep -oE '<!--[^>]*-->' admin_index.html <!-- VAULT-COMMENT: P3RoYXc9MjAyNS1SMS1PUklHSU4tS0VZ -->

The Sticky Note

The comment value is the hint — literally. P3R is the base64 fingerprint of ?t. Once you've seen it twice, you read the leading characters of base64 strings the way you read license plates.

$ echo 'P3RoYXc9MjAyNS1SMS1PUklHSU4tS0VZ' | base64 -d ?thaw=2025-R1-ORIGIN-KEY

The page told us the param name: thaw. The decoded comment confirmed it and supplied the value: 2025-R1-ORIGIN-KEY. The developers had thoughtfully embedded the entire query-string fragment in the comment, in case the player got confused about which part of the equals sign held the value.

Skipping the Door, Picking the Lock

The intended flow was: visit /admin/?thaw=2025-R1-ORIGIN-KEY, click "PRY THE DOOR," watch the React-y unlock animation. But the JS bundle gave away the actual mechanism:

// /admin/challenge-logic.js // 01 Freezer Aisle — Clue: VAULT-COMMENT decodes to `?thaw=<key>`. document.getElementById('cold-thaw-btn').addEventListener('click', async () => { const k = new URL(location.href).searchParams.get('thaw'); await handleUnlock('cold', 'freezer_aisle', k); }); // → POST /api/s2/web-hex/unlock {door, keyCandidate} // Server hashes, compares, returns flag in response.

One POST. No browser required. Skip the animation, hit the endpoint directly:

What I Learned

Three takeaways from the easiest door in a six-stage Web track:

HTML comments are not invisible. They are part of the response body, served to every requester, regardless of authentication. Treat them as load-bearing data. The first thing to do on any web page that smells like a challenge is grep '<!--' — or its equivalent in the inspector. Comments are where developers stash todo notes, debugging stubs, and, occasionally, override keys.

Read base64 by sight. The character set [A-Za-z0-9+/=], length divisible by 4, no whitespace, no separators. P3R at the start means the decoded text starts with ?t. SFRU means HTT (likely a URL). eyJ means {" (likely JSON, often a JWT). These three fingerprints alone cover most of what you'll see in the wild.

JS bundles are documentation. Pulling challenge-logic.js didn't just confirm the unlock flow — it pre-leaked the bug class for all five remaining stages of the Web track. Storage tier reassembly, JWT tier-field tampering, client-side redactor, prototype pollution, DOM XSS into window.vaultConfig.finalKey. The developers wrote a roadmap and shipped it next to the maze.
