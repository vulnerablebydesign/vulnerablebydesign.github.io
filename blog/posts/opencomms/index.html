The Brief That Was Three Riddles In One Sentence

"Every crew needs comms. The SnackCon relay has a Discord channel waiting for your signal. Find the terminal, open the link, and make first contact. Your first transmission is all it takes."

That sentence is the entire challenge:

  1. Find the terminal — not a generic shell. There is a specific SNACK_SHELL hidden in the platform, and the discord invite is only addressable from inside it.
  2. Open the link — once in SNACK_SHELL, run the command that prints the Discord invite URL.
  3. Make first contact — join the Discord, send any message. The server credits the solve via the chat/send endpoint.

This challenge has no CTF{...} flag. It's credit-only. The "flag" is the act of completion.

Step 1 — Find The Terminal

SNACK_SHELL is gated behind two DevTools-Console function calls. They aren't documented anywhere visible on the snack_con UI — you find them by reading the /season2/ page's snackSettings() introspection or by sniffing the bundled JS.

// in DevTools Console on https://snackspacecon.com/season2/ > enable1337SnackerMode() [1337 mode unlocked. Snackman game and 1337 text now available.] > enableTelnetMode() [Telnet mode active. SNACK_SHELL launching...]

The shell appears as a tmux-shaped pane overlay with status bar, panes, and a Linux-flavored prompt. It implements the usual coreutils (ls, cd, cat, grep, find) and a number of Season-2-specific commands (register, challenges, submit, stats, scoreboard, token, status) plus chat clients.

Step 2 — Open The Link

Inside SNACK_SHELL:

$ snackchat --discord 🍔 EmojiChef Encoding Active Your plain text is encoded by EmojiChef before sending — the 🍔 burger appearing in chat represents your encoded message, relayed through SnackBot. → discord.gg/[redacted] (Join the Snack_Con Discord →)

The shell prints the invite URL as a clickable link inside a system toast. Open it, accept, you're in the Snack_Con Discord server.

Step 3 — Make First Contact

The challenge is solved when a chat message is sent under your authenticated session. SnackChat auto-encodes outgoing messages with EmojiChef — your plaintext becomes a sequence of food emoji that SnackBot relays to the channel as a 🍔 burger indicator. (EmojiChef itself is a callback to S1R2, a previous round Chell won; her tooling has retroactively become canon.)

Sending the message hits POST /api/s2/chat/send. The server marks opencomms solved.

What I Learned

One thing, but it generalizes: read challenge briefs as compound sentences. "Find the terminal, open the link, make first contact" looks like one task. It's three. Each clause is its own subtask, and each subtask has its own discoverable hint trail in the platform's source. CTF authors compress puzzles into prose all the time — terminal meant SNACK_SHELL, link meant the Discord invite, first contact meant the chat-send. None of those mappings are obvious until you assume each noun is named in the source somewhere.

Attack Path Summary

  1. Open DevTools Console on /season2/
  2. enable1337SnackerMode() → enableTelnetMode() → SNACK_SHELL launches
  3. snackchat --discord → prints invite discord.gg/[redacted]
  4. Join the Discord; send any message in-channel via SnackChat
  5. POST /api/s2/chat/send credits the solve. No CTF{...} awarded — credit-only.
