What's Actually Different About Queso

The Tier 2 board is 9×9 with 10 mines, against Tortilla's 5×5/5. Same timeout (2000ms), same submission format, same scoring tracks. The taste-test list is a superset of Tortilla's eight, plus two new ones that explicitly probe the bigger pan:

  1. Corner cascade — reveal at (0, 0) on a zero-mine board fills the entire 9×9 grid via the BFS cascade. Tests that the cascade works from a corner, where each cell has only 3 neighbors instead of 8.
  2. Flag stress — 30 rapid flag toggles, alternating cells, must track minesRemaining exactly through every step.

The corner-cascade test is the interesting one. A naive implementation of cascade BFS could miss it if the engine's bounds-checking is too eager — e.g., if you skip enqueuing neighbors when any of nx, ny is at the edge of the board. The reference algorithm handles it correctly:

*_neighbors(cx, cy) { for (let dy = -1; dy <= 1; dy++) { for (let dx = -1; dx <= 1; dx++) { if (dx === 0 && dy === 0) continue; const nx = cx + dx, ny = cy + dy; if (this._inBounds(nx, ny)) yield [nx, ny]; } } }

From (0, 0) the generator yields [1, 0], [0, 1], [1, 1] — three valid neighbors, the rest filtered by _inBounds. The BFS proceeds. On a board with no mines, every cell has adjacentMines = 0, so every revealed cell enqueues its three-to-eight neighbors. The cascade fills the entire 81-cell grid in a single reveal(0, 0) call.

Flag stress is even less interesting if you've gotten test 4 right: my engine increments flagsPlaced on flag-true and decrements on flag-false. minesRemaining = mineCount - flagsPlaced works for any pattern of toggles, including duplicate flags on the same cell.

The Take-Away

This is the cheap-points part of the post: contract-conformant code carries you up the difficulty curve for free. The CHIPSWEEPER track is set up to reward this explicitly — if your engine passes Tier N's tests, the harness automatically attempts Tier N+1, and the kitchen credits the higher flag if it passes. There's no separate submission. There's no "I'll come back to that later." If you can pass the bigger pan's contract, you do.

The two new Queso tests (corner cascade, flag stress) are the kind of thing a sloppy implementation would miss. A textbook BFS cascade with proper bounds-checking via a generator helper, and a flag-toggle that increments/decrements a counter, both pass without needing any special-casing for tier 2.

What I Learned

If you're tempted to write a tier-1-specific solution and worry about generalizing later, don't. The cost of writing the general engine up front is roughly the same as the tier-specific one (you're already writing init(width, height, mineCount); the hard part is the first-bite-safety, which doesn't change with board size). The reward, in CHIPSWEEPER's case, was 500 free points and a flag I didn't aim at.

This generalization-is-cheap pattern is worth recognizing in coding-category challenges where the testing harness is tiered: the spec is the strategy. Cover the spec, the tiers fall.
