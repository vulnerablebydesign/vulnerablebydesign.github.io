The Mint Flow

The chain page's Stage 3 panel required two steps:

Step 1 — Mint a Receipt NFT. The form asked for the Stage 1 passphrase in plaintext: st0rag3_is_n3v3r_pr1vat3. Behind the scenes, the page called two server endpoints:

  • /api/s2/chain/nonce — a server-issued single-use nonce
  • /api/s2/chain/mint-auth — a signed authorization binding the wallet address to the registered player handle

The contract's mint() function enforces all of this cryptographically:

function mint( bytes32 passphrase, // Must match _secretPassphrase string playerName, // Registered Season 2 handle uint256 nonce, // Server-issued, single-use bytes signature // EIP-191 sig from trustedSigner ) external { require(passphrase == _secretPassphrase, "Wrong passphrase"); require(!_usedNonces[nonce], "Nonce used"); // Verify server signature binds: sender + name + nonce + contract + chainId bytes32 digest = keccak256(abi.encodePacked( msg.sender, playerName, nonce, address(this), block.chainid )).toEthSignedMessageHash(); require(digest.recover(signature) == trustedSigner, "Bad signature"); ... }

No shortcuts here. The passphrase must match, the signature must come from the trusted signer, and the nonce can only be used once. Even if someone intercepted the passphrase, they couldn't mint without the server's cryptographic authorization for their specific wallet address.

Step 2 — Verify Ownership. After the mint transaction confirmed on Sepolia (~16 seconds), clicking "Connect Wallet & Verify" triggered an on-chain ownership check via /api/s2/chain/verify-ownership. The server confirmed the connected wallet owned a SnackToken NFT, and the flag appeared.

The MetaMask confirmation dialog showed exactly what we expected: the mint function call with the passphrase (as bytes32), player name (her0x0ftime), a server nonce, and the cryptographic signature. Gas cost: 0.0002 SepoliaETH.

What I Learned

Stage 3 was the operational bookend to a cryptographic journey. The technical challenge was minimal — the hard work was done in Stages 1 and 2 — but it taught a different set of skills: wallet setup, testnet operations, understanding gas fees, and the mechanics of on-chain transactions.

The contract design was elegant. The trustedSigner pattern ensures that even with the passphrase leaked (which it is, by design — that's Stage 1), you can't forge a mint without server authorization. The nonce prevents replay attacks. The chain ID binding prevents cross-chain replay. Every security consideration in the mint() function exists for a real-world reason.
