The Sanitizer In Code

function sanitize(html) { const tmpl = document.createElement('template'); tmpl.innerHTML = html; const walk = (node) => { for (const c of Array.from(node.childNodes)) { if (c.nodeType !== 1) continue; const tag = c.tagName.toLowerCase(); if (['script','style','iframe','object','embed'].includes(tag)) { c.remove(); continue; } for (const attr of Array.from(c.attributes)) { const n = attr.name.toLowerCase(); if (n.startsWith('on') || n === 'style') c.removeAttribute(attr.name); if ((n === 'href' || n === 'src' || n === 'action') && /^\s*(javascript|data):/i.test(attr.value)) c.removeAttribute(attr.name); } walk(c); } }; walk(tmpl.content); return tmpl.innerHTML; } document.getElementById('clobber-verify-btn').addEventListener('click', async () => { const vc = window.vaultConfig; if (!vc) { setStatus('clobber', 'err', '...'); return; } const candidate = (vc.finalKey && (vc.finalKey.value ?? vc.finalKey)) || (typeof vc.getAttribute === 'function' ? vc.getAttribute('finalKey') : null); if (!candidate || typeof candidate !== 'string') return; await handleUnlock('clobber', 'buffet_counter', candidate); });

The sanitizer correctly strips <script>, <style>, <iframe>, <object>, <embed>, all on* handlers, all style attrs, and javascript:/data: URI schemes. What it does not strip: id and name attributes on regular HTML tags — which means any tag with id="vaultConfig" automatically becomes window.vaultConfig in HTML5 browsers (named-item access). And any name="finalKey" on a child becomes a property of that named element.

What I Learned

HTML5 named-item access is a legacy global namespace. Any <a id="X"> creates window.X. Any <form id="Y"><input name="Z"></form> creates window.Y.Z. Sanitizers that focus on script execution but allow id/name attributes are a structural miss; the bypass requires zero JavaScript. DOMPurify's SANITIZE_NAMED_PROPS option exists exactly for this. Vanilla allow-listed sanitizers usually don't.

Form elements are the cleanest clobber. <form> exposes its named children as direct properties — so vaultConfig.finalKey.value works in one nesting level without needing document.getElementsByName tricks. <input> is the cleanest carrier because its value attribute is a string and lots of code reads .value ?? ... on whatever it dereferences. Match the carrier to the dereferencer.

Cross-reference the metadata. "Current season theme" wasn't named on the inspector card. It came from a single line in LOG_EVENTS dated 2026-01-14.
